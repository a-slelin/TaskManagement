## Цели

Цель репозитория показать развитие некоторого проекта от самых "низов" (не используем Spring и частично JakartaEE) до
самых "верхов" (полностью написан на Spring Boot и использует микросервисную архитектуру).

## Описание

Проект версии - **v2.0**.

Этапы (добавления к версии 1.0):

1) Добавил простенький аудит в модель.
   ![Модель](images/model.svg)
2) Добавлен ORM через Hibernate.
3) Добавил валидацию через Hibernate Validator.
4) DAO работают теперь не напрямую с JDBC, а с EntityManager.
5) Исправил DTO (и Mapper) для более корректной отправки.

Улучшения:

1) Код (сущностей) стал гораздо легче читать, что-то добавить уже не так тяжело.
2) На уровне кода проверяется согласованность данных (телефон должен удовлетворять паттерну, например).
3) DAO писать вполне легко, из-за простого JPA интерфейса (не надо сто раз открывать connection).

Проблемы (для будущей доработки):

1) Очень неудобно постоянно открывать и закрывать транзакции (em.getTransaction().begin()), вот бы кто-нибудь их сам
   обрабатывал, а мы декларативно указывали где она должна быть.
2) Опять синглтоны... Вот бы кто-то создал контекст и вставлял нам нужные объекты.
3) Сервлеты содержат много однотипного кода, тяжело воспринимать код.

Ожидаемые обновления:

В версии **v3.0** ожидается переход на использование Java API для веб-сервисов RESTful - JAX-RS.

## Требования

* Для запуска необходимо предустановленная СУБД Postgres (я использую версию **18.3**), либо она же запущенная с помощью
  docker.
* Также необходимо скачать Apache Tomcat контейнер сервлетов (я использую версию **11.0.18**).
* Apache Maven - с помощью него будет собран проект (использую версию **3.9.9**).

## Запуск

Для того чтобы запустить приложение, необходимо:

1) Запустить Postgres СУБД нативно или же через докер-контейнер (docker/docker-compose.yaml):

   ````shell
   cd docker;
   docker compose up;
   ````

2) Подключиться к СУБД и выполнить sql-скрипт [`init.sql`](src/main/resources/db/init.sql) по созданию базы данных и
   пользователя для подключения к ней.

3) Далее необходимо выполнить последовательно два скрипта: [`schema.sql`](src/main/resources/db/schema.sql) - для
   создания схемы в базе данных и [`data.sql`](src/main/resources/db/data.sql) - для создания тестовых данных.

4) Запустить контейнер сервлетов Tomcat на порту **8080** с помощью готового скрипта **startup.bat**:

   ````shell
   cd <ваш путь к Tomcat>;
   .\startup.bat;
   ````

5) Клонируем проект, собираем его и устанавливаем в локальный репозиторий maven (.m2):

   ```shell
   git clone https://github.com/a-slelin/TaskManagement.git;
   cd TaskManagement;
   mvn clean install;
   ```

6) Проект запущен на вашем локальном Tomcat!!! Можно перейти в подходящий HttpClient (Insomnia, PostMan, CLI) и работать
   с приложением как вам нужно.
