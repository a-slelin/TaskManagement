# Система управления задачами и уведомлений

## Цели

Цель репозитория показать развитие некоторого проекта от самых "низов" (не используем Spring и частично JakartaEE) до
самых "верхов" (полностью написан на Spring Boot и использует микросервисную архитектуру).

## Описание

Проект версии - **v1.0**.

Этапы:

1) Создана простенькая модель данных (используем lombok для ускорения работы).
   ![Модель](images/model.svg)
2) Созданы DAO-классы, чтобы показать низкоуровневые обращения к базе данных с помощью JDBC.
3) Созданы DTO-классы для общения через сеть.
4) Созданы Mapper-классы для сопоставления entity-dto и наоборот (используем mapstruct для ускорения работы).
5) Созданы сервисы.
6) Созданы сервлеты для общения через REST API, которые вызывают сервисы. Общение осуществляется с помощью json
   (библиотека jackson).

Получили простое веб-приложение которое может на низком уровне работать с CRUD-операциями по сети с помощью
архитектурного стиля REST. Проблемы данного веб-приложения очевидны:

1) Очень тяжелая логика поддержки работы с базой данных, также трудно что-то менять.
2) User-NOT-friendly сервлеты: если что-то не так сразу ошибка 500 и сиди, разбирайся почему.
3) Тонны кода, который из раза в раз повторяется.
4) Очень тяжело улучшать проект, очевидно, что дальше мы станем заложниками кода, написанного до нас, что сильно
   ограничивает.
5) Постоянное использование паттерна Singleton: это выглядит (и подвержено ошибкам) очень странно, тогда как было бы
   хорошо, чтобы кто-то создавал контекст и следил за ним.
6) Другие проблемы.

Ожидаемые обновления:

В версии **v2.0** ожидается переход на использование популярной реализации JPA - Hibernate.

## Требования

* Для запуска необходимо предустановленная СУБД Postgres (я использую версию **18.0**), либо она же запущенная с помощью
  docker.
* Также необходимо скачать Apache Tomcat контейнер сервлетов (я использую версию **11.0.18**).
* Apache Maven - с помощью него будет собран проект (использую версию **3.9.9**).

## Запуск

Для того чтобы запустить приложение, необходимо:

1) Запустить Postgres СУБД нативно или же через докер-контейнер (docker/docker-compose.yaml):

   ````shell
   cd docker;
   docker compose up;
   ````

2) Подключиться к СУБД и выполнить sql-скрипт [`init.sql`](src/main/resources/db/init.sql) по созданию базы данных и
   пользователя для подключения к ней.

3) Далее необходимо выполнить последовательно два скрипта: [`schema.sql`](src/main/resources/db/schema.sql) - для
   создания схемы в базе данных и [`data.sql`](src/main/resources/db/data.sql) - для создания тестовых данных.

4) Запустить контейнер сервлетов Tomcat на порту **8080** с помощью готового скрипта **startup.bat**:

   ````shell
   cd <ваш путь к Tomcat>;
   .\startup.bat;
   ````

5) Клонируем проект, собираем его и устанавливаем в локальный репозиторий maven (.m2):

   ```shell
   git clone https://github.com/a-slelin/TaskManagement.git;
   cd TaskManagement;
   mvn clean install;
   ```

6) Проект запущен на вашем локальном Tomcat!!! Можно перейти в подходящий HttpClient (Insomnia, PostMan, CLI) и работать
   с приложением как вам нужно.
